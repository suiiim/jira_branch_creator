# 커밋 메시지 규칙 (Commit Message Convention)

## 기본 형식

```
<type>: <제목>

[본문 - 선택]
```

## 커밋 타입 목록

| 타입       | 설명                              | 사용 예 |
|------------|-----------------------------------|--------|
| `feat`     | 새로운 기능 추가                   | 새 API 연동, 신규 화면 |
| `fix`      | 버그 수정                         | 예외 처리, 로직 오류 수정 |
| `style`    | UI/스타일 변경 (기능 무관)         | CSS, 레이아웃 변경 |
| `refactor` | 리팩토링 (기능/버그 변경 없음)     | 함수 분리, 변수명 변경 |
| `docs`     | 문서 수정                         | README, 주석 업데이트 |
| `chore`    | 설정, 패키지 등 기타 작업          | .gitignore, 패키지 업데이트 |
| `remove`   | 파일/코드 삭제                    | 미사용 파일 제거 |
| `wip`      | 작업 중 (임시 커밋)               | 미완성 상태 임시 저장 |

## 작성 규칙

1. **언어**: 한국어로 작성
2. **형식**: `타입: 제목` (콜론 + 공백 필수)
3. **마침표**: 제목 끝에 마침표 없음
4. **길이**: 제목 50자 이내 권장
5. **어조**: 명령형 또는 명사형 → "추가함" ❌ / "추가" ✅

## 올바른 예시

```bash
# 새 기능
git commit -m "feat: SSCVE 이슈에서 브랜치 자동 생성 기능 추가"

# 버그 수정
git commit -m "fix: 한글 이슈 요약 시 브랜치명 생성 오류 수정"

# 문서
git commit -m "docs: README 환경변수 설정 방법 업데이트"

# 리팩토링
git commit -m "refactor: 브랜치 네이밍 로직 함수 분리"

# 설정 변경
git commit -m "chore: Python 의존성 패키지 버전 업데이트"

# 파일 삭제
git commit -m "remove: 미사용 레거시 스크립트 제거"
```

## 잘못된 예시

```bash
# ❌ 마침표 있음
git commit -m "feat: 브랜치 생성 기능 추가."

# ❌ 타입 없음
git commit -m "브랜치 생성 기능 추가"

# ❌ 동사형 종결어미
git commit -m "feat: 브랜치 생성 기능을 추가했습니다"

# ❌ 너무 모호함
git commit -m "fix: 버그 수정"
```
