# Commit Message Convention

## 기본 형식

```
<type>: <제목>

[본문 - 선택]
```

## 타입 목록

| 타입 | 설명 | 예시 |
|------|------|------|
| `feat` | 새로운 기능 추가 | `feat: 로그인 기능 추가` |
| `fix` | 버그 수정 | `fix: 비밀번호 유효성 검사 오류 수정` |
| `style` | UI/스타일 변경 (기능 무관) | `style: 버튼 색상 변경` |
| `refactor` | 리팩토링 (기능/버그 변경 없음) | `refactor: 유저 조회 로직 분리` |
| `docs` | 문서 수정 | `docs: README 설치 방법 업데이트` |
| `chore` | 설정, 패키지 등 기타 작업 | `chore: eslint 설정 추가` |
| `remove` | 파일/코드 삭제 | `remove: 사용하지 않는 컴포넌트 삭제` |
| `wip` | 작업 중 (임시 커밋) | `wip: 결제 페이지 작업 중` |

## 제목 작성 규칙

- 한국어로 작성
- 타입 뒤에 콜론 + 공백 → `feat: `
- 마침표 없이 끝내기
- 50자 이내 권장
- 명령형 또는 명사형으로 작성 → "추가함" ❌ / "추가" ✅

## 본문 작성 규칙 (선택)

- 제목 아래 한 줄 띄고 작성
- 무엇을, 왜 변경했는지 설명 (어떻게는 코드로 확인 가능)
- 토이 프로젝트에서는 생략해도 무방

## 예시

```
feat: 할 일 추가 기능 구현

LocalStorage에 데이터 저장하도록 처리
```

```
fix: 날짜 포맷 오류 수정
```

```
chore: tailwind CSS 패키지 설치
```

```
wip: 검색 필터 기능 작업 중
```

## 자주 하는 실수 ❌

| 나쁜 예 | 좋은 예 |
|---------|---------|
| `수정함` | `fix: 로그인 버튼 클릭 오류 수정` |
| `작업중` | `wip: 메인 페이지 레이아웃 작업 중` |
| `feat: 기능 추가.` | `feat: 다크모드 기능 추가` |
| `fix: 버그` | `fix: 새로고침 시 상태 초기화 버그 수정` |
